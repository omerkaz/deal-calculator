## Post-Ship Hardening (2026-07-06) — IMPORTANT

The milestone verdict had been overridden to pass; runtime verification later
found the automation layer shipped broken. Fixed and verified end-to-end:

| Bug | Fix |
|-----|-----|
| `PRACTITIONER_USER_ID` Edge env never set → every webhook/email call 500'd | Functions fall back to `practitioner_settings` row (D015); deployed webhook v5, send-email v2 |
| `pg_net.http_post` called in 6 cron functions (schema is `net.`) | `net.http_post`; migration `fix_cron_email_functions` |
| Lifecycle reminders sent `patient_*` payload; send-email needs `to/subject/html` | Payloads rebuilt from src/lib/email.ts templates |
| Functions URL from unsettable GUC (always NULL in repo schema) | Vault secret `SUPABASE_FUNCTIONS_URL` (D016) |
| Reminders re-fired daily forever past threshold | 24h BETWEEN windows + COALESCE(state_changed_at, created_at) (D017) |
| auto-cold raced day-12 email at 09:00 | auto-cold moved to 10:00 UTC (D018) |
| `practitioner_settings` row missing → all emails silently skipped | Row provisioned (all toggles false/opt-in) |
| Login never navigated on success | `navigate("/")` after signIn |

**Verification evidence:** webhook 201 with correct attribution; cron function
→ `net._http_response` 200 with Resend `sent:true`; toggle-off negative test:
zero dispatches; login → dashboard → settings verified in browser.
