# Milestone v1.0: Core Patient Management CRM

**Status:** ✅ SHIPPED (as GSD-3 milestone M001)
**Phases:** 1–6 (GSD-3 slices S01–S06)

## Overview

Full patient CRM: Supabase schema with RLS, auth, patient CRUD with search and
filters, 9-state lifecycle machine with optimistic concurrency, manual payment
recording with derived status, notes and file attachments, dashboard with
pipeline metrics, and Kanban pipeline view. 116 modules, 0 TS errors, 12
node:test assertions on dashboard math.

## Phases (reconstructed)

1. **Schema + Auth** — 4 tables with RLS, updated_at trigger, AuthProvider with INITIAL_SESSION gate
2. **Patient CRUD** — list/search/filter, dual-mode form, data-access `{ data, error }` pattern
3. **Lifecycle machine** — VALID_TRANSITIONS map, transitionState with optimistic concurrency
4. **Payments** — recording + derived paid/partial/unpaid vs PACKAGE_PRICES
5. **Notes + attachments** — Storage bucket `patient-files`, signed URLs, upload rollback
6. **Dashboard + Pipeline** — computeMetrics() pure function, Kanban columns
