<objective>
Rewrite all 6 reminder cron functions to replace the fragile 24h BETWEEN windows
with a durable "due AND not yet sent" pattern using the email_send_log table
from plan 14-01. This is the core MAIL-02 deliverable.

Purpose: Ensure at-least-once delivery — if a cron run is missed, the reminder
catches up on the next run instead of being permanently skipped.

Output: Updated cron functions in schema.sql + migration SQL with the rewrites.

Query Pattern (replaces BETWEEN):
Instead of:
  WHERE COALESCE(state_changed_at, created_at)
        BETWEEN now() - INTERVAL '15 days' AND now() - INTERVAL '14 days'

Use:
  WHERE COALESCE(state_changed_at, created_at) <= now() - INTERVAL '14 days'
    AND NOT EXISTS (
      SELECT 1 FROM email_send_log esl
       WHERE esl.patient_id = patients.id
         AND esl.feature = 'blood_test_reminder'
         AND esl.sent_at >= COALESCE(patients.state_changed_at, patients.created_at)
    )

The sent_at >= state_changed_at comparison ensures re-engagement correctness:
if a patient goes cold → lead again, state_changed_at resets, and old log
entries from the prior lifecycle run don't count as "already sent."
</objective>
